名詞小百科（講給組員與評審聽）

點開展開。本週的關鍵概念都跟「證明系統有用」有關。

＋準確率曲線是什麼？為什麼它是簡報的核心證據？實驗設計

我們的主張是「人標一點點，系統標剩下的」。要證明它，就要回答：種子標到第幾個，系統就夠準了？

做法：固定一批圖和正解，種子從 0 個逐步加到 20 個，每加一個就量兩個數字——驗證準確率（系統標得對不對）和 auto_ratio（多少比例不用人看）。畫成曲線後，理想的形狀是「前幾個種子急速上升、之後趨平」——那個「趨平的點」就是我們的賣點：標 5 個，省 70% 工時這種一句話結論，就是從這張圖讀出來的。

seed=0 → 全部送審（0% 省工時）→ seed=3 → 準確率 80%、省 60% → seed=10 → 趨平 ← 這裡就是證據
＋特徵快取為什麼讓主動學習「即時」？效能

主動學習迴圈是：標一個種子 → 回訓分類器 → 重新預測所有未審片段。重新預測需要每個片段的特徵向量——但向量根本沒變（圖和遮罩都沒動），變的只有分類器。

現在的寫法每次都重跑 DINOv2 去算向量，真模型下標一個種子要等幾十秒。改成第一次算完就存起來之後，回訓只重跑 kNN 預測（毫秒級）——「標一個種子、滿畫面顏色即時翻新」這個 demo 最有感的瞬間才成立，佳凱的實驗（要重複幾百次）也才跑得完。

改前：標種子 → 全部重跑 DINOv2（幾十秒）❌　　改後：標種子 → 讀快取 → 只重跑 kNN（<0.1s）✅
＋SAM 為什麼慢？提速在調什麼？效能

「自動分割整張」的原理是在圖上撒一格提示點（points_per_side=32 就是 32×32＝1024 個點），每個點都問一次「這裡有什麼」，再把上千個候選遮罩去重過濾。慢的通常不是模型推論，而是點太多＋後處理在 CPU 上跑。

兩個旋鈕：點撒少一點（32→16，剩 1/4）、圖縮小一點再切（1024→768）。代價是小物件可能漏切，所以要跑對照表用數據決定。另外「點一下切一塊」和 Grounding DINO 的框提示本來就只問一兩個點，不受影響——素人流程天生是快的。

＋一鍵訓練證明了什麼？閉環

標註平台的終點不是「標完」，是標出來的資料真的能訓練出模型。一鍵訓練腳本把「匯出 YOLO 資料集 → 訓練 YOLOv8-seg → 出 mAP 和預測圖」串成一條命令。

對評審來說這是完整的故事：素人打一句話 → 系統標好 → 匯出 → 訓練出一個能認貓的模型——從原始照片到可用模型，全程在我們的平台裡發生。Week 4 的成果數據（mAP、預測範例圖）就從這裡產出。

標註（本平台）→ export YOLO → ultralytics 訓練 → mAP + 預測圖（簡報素材）
