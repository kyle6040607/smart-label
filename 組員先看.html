📋 項目概述

核心目標

打造一個完整的端對端 AI 標注和訓練平台：

  • 自動分割 → SAM + Grounding DINO
  • 人工審核 → 主動學習篩選
  • 一鍵訓練 → YOLOv8 模型
  • 雲端部署 → Google Cloud Platform
核心價值： 用 AI 輔助標注，省時省力。一個平台完成從標注→訓練→部署的全流程

🛠️ 技術棧

🧠 AI/ML

  • SAM - 圖像分割
  • Grounding DINO - 文字驅動
  • DINOv2 - 特徵提取
  • YOLOv8 - 物件偵測
  • 主動學習引擎

🖥️ 平台技術

  • Python 3.13 + Flask
  • HTML5 Canvas + Vanilla JS
  • Chart.js 數據可視化
  • Docker 容器化
  • Google Cloud Run

📍 Week 1 - 環境搭建

目標：所有人本地跑通，A 定好輸出格式讓 B、C 同步開工

組員 主要任務 交付物
組長 系統架構設計、API 介面規格定義 架構文件 + API 介面規格
組員 A SAM mock 跑通、輸出遮罩格式定義 mock segmenter + 格式文件
組員 B 前端改版、確認 A 的遮罩格式 index.html 改進版
組員 C Docker 環境建好、YOLOv8 環境準備 Dockerfile + 環境確認報告
📚 推薦學習： SAM 論文、DINOv2 論文、YOLOv8 文檔、Docker 官方教程

📍 Week 2 - AI 核心功能

目標：三條 ML 線路各自跑通，C 用 mock 資料先跑 YOLOv8

組員 主要任務 交付物
組長 Code Review、模組整合協調 Code Review 紀錄
組員 A Grounding DINO 整合 Grounding DINO Pipeline
組員 B DINOv2 特徵提取 + KNN 分類器 特徵模組 + 分類器
組員 C YOLOv8 pipeline（mock 資料先跑通） YOLOv8 pipeline + 測試報告
📚 推薦學習： Grounding DINO 論文、DINOv2 文檔、YOLOv8 訓練流程、KNN 原理

📍 Week 3 - 整合 & 部署

目標：三條線匯流，全流程端對端跑通，GCP 上線

組員 主要任務 交付物
組長 整合全流程、效能調優 整合測試報告 + 部署清單
組員 A SAM + Grounding DINO Benchmark 實驗 效能比較報告
組員 B Dashboard 圖表、主動學習流程串接 完整 Dashboard
組員 C 真實標注資料訓練 YOLOv8、GCP 部署 訓練結果報告 + GCP 部署
📚 推薦學習： Chart.js、Google Cloud Run、pytest、效能測試

📍 Week 4 - 衝刺&展示

目標：完成文檔、最後測試、排練展示

組員 主要任務 交付物
組長 展示排練主持、最終整合、簡報準備 展示腳本 + 系統架構圖
組員 A ML 技術文檔、實驗結果整理 技術文檔 (3-5 頁)
組員 B UI/UX 設計文檔、最後微調 設計文檔 (2-3 頁)
組員 C 部署文檔、故障排除指南 部署文檔 (3-4 頁)

📢 展示排練 (30-40 分鐘)： 各人 8-10 分鐘發言 + Q&A
